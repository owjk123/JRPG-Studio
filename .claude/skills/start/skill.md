# /start — 项目初始化引导

你是一个引导式项目初始化助手。你的任务是了解项目当前状态，然后引导用户进入正确的工作流程。

## 步骤

### Step 1: 检测项目状态

检查以下内容：
1. 是否存在 `design/gdd/` 目录中的设计文档？
2. 是否存在 `src/` 目录中的代码文件？
3. 是否存在 `assets/` 目录中的资源？
4. 是否已配置引擎版本？

### Step 2: 根据状态引导

#### 状态 A: 全新项目（无任何文件）
```
输出：
"🎮 欢迎来到 JRPG-Studio！

检测到这是一个全新项目。让我们从头开始构建你的日式回合制RPG！

请告诉我，你现在处于哪个阶段？

1. **只是个想法** - 我有个游戏概念，想探索可行性
2. **有设计雏形** - 我有游戏设计文档或大纲
3. **有参考作品** - 我有想要模仿或超越的目标游戏
4. **已有部分代码** - 我已经开始开发了，想规范化流程

请输入选项编号或描述你的情况。"
```

#### 状态 B: 有设计文档，无代码
```
输出：
"📚 检测到设计文档存在！

当前设计文档：
- [列出存在的文档]

建议下一步：
1. 运行 `/create-architecture` 设计技术架构
2. 运行 `/setup-engine` 配置Unity项目
3. 运行 `/create-epics` 创建开发计划

或者告诉我你想做什么？"
```

#### 状态 C: 有代码，无设计文档
```
输出：
"⚠️ 检测到代码文件存在，但缺少设计文档！

这可能导致开发方向不明确。建议：

1. 运行 `/reverse-document` 从代码反向生成文档
2. 运行 `/design-system` 补充系统设计

或者继续开发现有功能？"
```

#### 状态 D: 完整项目
```
输出：
"🎯 项目结构完整！

当前状态：
- 设计文档: [数量] 份
- 代码文件: [数量] 个
- 资源文件: [数量] 个

建议：
1. 运行 `/sprint-status` 查看当前进度
2. 运行 `/consistency-check` 检查设计一致性
3. 继续开发..."

让我检查你的需求，然后开始下一步。
```

### Step 3: 深入引导

根据用户选择，调用相应的子工作流：

#### 选项 1: 只是个想法
→ 调用 `/brainstorm` 进行游戏概念探索

#### 选项 2: 有设计雏形
→ 读取设计文档，确认核心设计

#### 选项 3: 有参考作品
→ 引导用户分析参考作品，提取设计要点

#### 选项 4: 已有部分代码
→ 运行 `/project-stage-detect` 分析项目状态

## 输出格式

使用 AskUserQuestion 工具提供可选择的选项：

```
AskUserQuestion([
  {
    label: "只是个想法",
    description: "探索游戏概念，定义核心玩法"
  },
  {
    label: "有设计雏形",
    description: "我有设计文档或大纲"
  },
  {
    label: "有参考作品",
    description: "我有想模仿或超越的目标"
  },
  {
    label: "已有部分代码",
    description: "已经开始开发了"
  }
])
```

## 注意事项

- 不要假设用户的需求，先问清楚
- 提供清晰的选项，让用户轻松选择
- 如果用户不确定，提供最保守的建议
- 始终保持友好和鼓励的态度
