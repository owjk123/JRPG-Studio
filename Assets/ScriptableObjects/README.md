# JRPG-Studio 演示场景和测试数据

本文档说明为 JRPG-Studio 项目创建的演示场景和测试数据。

## 目录结构

```
Assets/ScriptableObjects/
├── Battles/              # 战斗遭遇配置
│   ├── DemoBattle1_NewbieForest.asset
│   ├── DemoBattle2_GoblinCamp.asset
│   ├── DemoBattle3_DarkCastle.asset
│   └── EnemyDataConfig.cs
├── Tutorial/              # 新手引导配置
│   ├── TutorialStep.cs
│   └── TutorialData.asset
├── Shop/                  # 商店配置
│   ├── ShopItemData.cs
│   ├── DailyShop.asset
│   └── PremiumShop.asset
├── Player/                # 玩家存档模板
│   ├── NewPlayerTemplate.cs
│   └── NewPlayerTemplate.asset
└── Stages/                # 关卡配置
    ├── StageData.cs
    ├── Chapter1_Stage1.asset ~ Chapter1_Stage5.asset
    └── Chapter2_Stage1.asset ~ Chapter2_Stage5.asset
```

## 创建的文件详情

### 1. 战斗遭遇配置 (Battles/)

| 文件名 | 描述 | 敌人配置 |
|--------|------|----------|
| DemoBattle1_NewbieForest.asset | 新手森林战斗 | 3只Lv.1史莱姆 |
| DemoBattle2_GoblinCamp.asset | 哥布林营地 | 5只Lv.3-4哥布林 |
| DemoBattle3_DarkCastle.asset | 黑暗城堡BOSS战 | 骷髅兵、黑暗骑士、BOSS |

**战斗数值平衡：**
- 新手森林：推荐等级1，消耗体力6，奖励EXP50/金币100
- 哥布林营地：推荐等级5，消耗体力10，奖励EXP150/金币250
- 黑暗城堡：推荐等级18，消耗体力20，奖励EXP800/金币1000

### 2. 新手引导配置 (Tutorial/)

**TutorialStep.cs** - 定义引导步骤结构：
- 步骤ID和名称
- 引导类型（点击、拖拽、等待、战斗等）
- 目标路径和高亮样式
- 对话配置
- 条件配置

**TutorialData.asset** - 5步新手引导流程：
1. 介绍主界面（货币显示）
2. 角色编组
3. 首次战斗
4. 抽卡系统
5. 角色养成

### 3. 商店配置 (Shop/)

**ShopItemData.cs** - 商品数据结构：
- 商品类型（角色、装备、材料、消耗品等）
- 价格和货币类型
- 库存和限购配置
- 折扣标签

**DailyShop.asset** - 每日商店：
- 经验药水、种子包、灵石等养成材料
- 使用金币购买

**PremiumShop.asset** - 高级商店：
- 角色碎片、愿望星、宝箱
- 使用钻石购买

### 4. 玩家存档模板 (Player/)

**NewPlayerTemplate.cs** - 新玩家初始数据：
- 初始货币：10000金币 + 500钻石
- 初始角色：人族骑士（ID: 1001）
- 初始装备：武器、防具、饰品各一件
- 体力系统配置
- 默认游戏设置

### 5. 关卡配置 (Stages/)

### 经验值曲线

- 每级所需经验：100 × 等级
- 击败敌人获得经验：敌人基础EXP × 等级系数

## 使用说明

1. 在Unity编辑器中打开项目
2. ScriptableObject文件位于 `Assets/ScriptableObjects/` 目录下
3. 可直接在Unity Inspector中编辑各项数值
4. 战斗系统通过引用这些ScriptableObject来加载配置

## 注意事项

- `.asset` 文件中的 GUID 占位符需要根据实际脚本生成的 GUID 替换
- 敌人技能ID（1001-1019）为示例ID，需关联实际技能配置
- 剧情对话ID需关联实际对话配置文件
